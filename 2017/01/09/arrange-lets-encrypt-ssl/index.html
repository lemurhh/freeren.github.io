部署Let's Encrypt免费SSL证书

发表于 2017-01-09 | 分类于 随笔 |

一年前，我曾经花了上千大洋购买OV级SSL证书并且被SSL证书部署的麻烦虐出血。这次网站变更我对上次的惨痛经历仍心存余悸，故起先并未打算再次部署SSL证书。但后来发现HTTPS成为主流的趋势越来越明显，谷歌等厂商今年开始给不使用https的网站标注不安全标记，并且根据firefox的统计数据，过去一年浏览器加载的https网页比率从39%增长到49%。最终，我还是决定部署SSL证书了。这次选择的是免费的Let’s Encrypt。

Let’s Encrypt是由ISRG（Internet Security Research Group）提供的免费SSL项目，现由Linux基金会托管，他的来头很大，由Mozilla、思科、Akamai、IdenTrust和EFF等组织发起，现在已经得到Google、Facebook等大公司的支持和赞助，目的就是向网站免费签发和管理证书，推动HTTPS的普及。通过其自身的自动化过程，消除了购买、安装证书的复杂性，只需几行命令，就可以完成证书的生成并投入使用，甚至十几分钟就可以让自己的http站点华丽转变成Https站点。

准备

根据官方要求，在部署之前，需要系统支持python 2.7以上版本及git工具。对于centos系统，默认没有git工具，需要添加epel源后，自己安装。添加epel源还是很简单的。

[root@localhost ~]# wget http://dl.fedoraproject.org/pub/epel/epel-release-latest-7.noarch.rpm
[root@localhost ~]# rpm -ivh epel-release-latest-7.noarch.rpm
[root@localhost ~]# yum repolist      ##检查是否已添加至源列表
添加好源后，安装git就可以了。 
[root@localhost ~]# yum -y install git-core

获取SSL证书

Let’s Encrypt的自动化获取证书还是很方便的。

git clone https://github.com/letsencrypt/letsencrypt
cd letsencrypt
./letsencrypt-auto certonly --standalone --email admin@example.com -d example.com -d www.example.com
注：网站域名的DNS服务器最好不要选择国内的免费DNS，否则可能出现无法确认域名信息（The server could not connect to the client to verify the domain）的错误。 ## SSL证书部署 证书获取成功后，相应的证书文件位于/etc/letsencrypt/live/example.com/目录下，有4个。 cert.pem - Apache服务器端证书 chain.pem - Apache根证书和中继证书 fullchain.pem - Nginx所需要ssl_certificate文件 privkey.pem - 安全证书KEY文件 如果是nginx，只需要最后两个文件，找到网站的nginx配置文件，在其中添加如下配置信息即可。 
listen 443 ssl;
ssl_certificate /etc/letsencrypt/live/example.com/fullchain.pem;
ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;
为了能使通过http的访问也转向https，需要再对配置文件做一些修改。添加如下配置信息： 
if ($scheme = http) {
        return     301 https://$host$request_uri;
 }
再在location ~ \.php$ 块中添加： 
fastcgi_param  HTTPS  $https if_not_empty;
这样部署就算完成了。 ## SSL证书续期 Let's Encrypt证书的有效期是90天，到期后需要手工续期。 
./letsencrypt-auto certonly --renew-by-default --email admin@example.com -d example.com -d www.example.com
