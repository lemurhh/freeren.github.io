linux命令——mkdir、rmdir命令

发表于 2016-08-16 | 分类于 操作系统 |

创建目录命令——mkdir

在linux创建新目录不能使用touch命令，应使用mkdir命令。使用该命令系统将创建一个新的目录并分配一个新的索引节点号。

mkdir test #在当前目录下创建test目录

删除目录命令——rmdir

除了前面介绍的使用rm -r命令可以递归地删除目录外，也可以使用mkdir命令。但默认情况下该命令只删除空目录，否则将会给出错误提示。为删除非空目录，可以使用”–ignore-fail-on-non-empty”参数。

rmdir --ignore-fail-on-non-empty test #多字母参数要使用"--"而不是"-"

linux命令——mv、rm命令

发表于 2016-08-15 | 分类于 操作系统 |

移动和重命名文件——mv命令

在linux中移动和重命名文件和目录可以使用mv命令。移动文件可能会更改文件名，但会保留原来的索引节点号和时间戳。移动只带有硬链接(与文件完全相同的副本)的文件没有问题，但对于带有软链接的文件，会导致软链接指向一个无效的文件，从而不再是一个有效的链接。使用mv命令移动目录不会改变目录的结构，只是目录的名称发生了变化。

cp -l test test1
cp -s test test2
mv test test3 #test被重命名为test3，此时test1没有任何问题，但test2会成为一个无效链接
mv test3 test #test2恢复正常

可以通过上面命令结合ls -sil命令，检验一下上一篇文章中关于两种链接的叙述。

删除文件——rm命令

删除文件是一个常见的操作。

基本格式

rm [-options] file

该命令通常会让您确认是否删除文件，一旦通过该命令删除文件后将无法还原(bash shell中没有回收站)。rm命令也可以使用通配符。

删除带链接的文件

删除原始文件后，硬链接仍可以正常访问，但软链接将会失效。事实上，硬链接的索引节点与原始文件一样，在移除最后一个硬链接之前，硬链接将一直维持着原始文件的索引节点号，并在硬链接中保留原始文件的数据。而所有原始文件的变化对软链接产生的影响，包括但不限于删除原始文件，则应把握这样一个原则：原始文件不在了（包括被重命名被移动被删除），链接就会失效，就像win中的快捷方式。

常用参数

-f 删除前不提示确认
-r 递归地删除目录

linux命令——文件链接和ln命令

发表于 2016-08-15 | 分类于 操作系统 |

今天说一下文件链接。

文件链接

如果需要在系统中维护一个文件的多个副本，那么不一定需要使用多个物理副本，也可以使用多个虚拟副本来代替物理副本，这种虚拟副本就被称为链接。链接是目录中的占位符，指向文件的真实位置，类似于指针。linux中有两种链接类型：硬链接和符号链接(软连接)。

硬链接

硬链接创建一个单独的文件，里面内容和原始文件一样，大小与原始文件相同，引用硬链接的效果和引用原始文件一样。硬链接和原始文件拥有相同的索引节点，这表明实际上它们是同一个文件，也可以认为是一个与文件完全相同的副本，-l参数可以创建一个硬链接。

cp -l test test1

符号链接

符号链接只包含有关原始文件的信息，不存储原始文件中的实际数据，所以一般大小比原始文件小。符号链接和原始文件的索引节点不同，这说明这是两个不同的文件，-s参数可以创建一个符号链接。

cp -s test test2

ln命令

除了使用cp命令创建链接，还可以使用ln命令，ln命令默认创建硬链接，如果要创建符号链接还需要使用-s参数。

注意

不要复制链接文件，可以再创建一个原始文件的链接。对同一个文件生成多个链接没有问题，但不要创建链接文件的链接文件，这会产生一个链接链条，容易出现各种问题。

httpscan爬虫式主机发现工具

发表于 2016-08-13 | 分类于 渗透测试 |

简介

httpscan是一个扫描指定网段的Web主机的小工具。和端口扫描器不一样，httpscan是以爬虫的方式进行Web主机发现，因此相对来说不容易被防火墙拦截。httpscan会返回IP地址,http状态码,Web容器版本以及网站标题。

使用

./httpscan IP/CIDR -t threads #可使用CIDR扫描一个段的IP地址
#例子
./httpscan 192.168.10.1/24 -t 10

项目地址

https://github.com/zer0h/httpscan

主要代码

#!/usr/bin/env python
#coding:utf-8
# Author: Zeroh

import re
import sys
import Queue
import threading
import optparse
import requests
from IPy import IP

printLock = threading.Semaphore(1)  #lock Screen print
TimeOut = 5  #request timeout

#User-Agent
header = {'User-Agent' : 'Mozilla/5.0 (Windows NT 6.1; WOW64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/44.0.2403.125 Safari/537.36','Connection':'close'}

class scan():

  def __init__(self,cidr,threads_num):
    self.threads_num = threads_num
    self.cidr = IP(cidr)
    #build ip queue
    self.IPs = Queue.Queue()
    for ip in self.cidr:
      ip = str(ip)
      self.IPs.put(ip)

  def request(self):
    with threading.Lock():
      while self.IPs.qsize() > 0:
        ip = self.IPs.get()
        try:
          r = requests.Session().get('http://'+str(ip),headers=header,timeout=TimeOut)
          status = r.status_code
          title = re.search(r'<title>(.*)</title>', r.text) #get the title
          if title:
            title = title.group(1).strip().strip("\r").strip("\n")[:30]
          else:
            title = "None"
          banner = ''
          try:
            banner += r.headers['Server'][:20] #get the server banner
          except:pass
          printLock.acquire()
          print "|%-16s|%-6s|%-20s|%-30s|" % (ip,status,banner,title)
          print "+----------------+------+--------------------+------------------------------+"

          #Save log
          with open("./log/"+self.cidr.strNormal(3)+".log",'a') as f:
            f.write(ip+"\n")

        except Exception,e:
          printLock.acquire()
        finally:
          printLock.release()

  #Multi thread
  def run(self):
    for i in range(self.threads_num):
      t = threading.Thread(target=self.request)
      t.start()

if __name__ == "__main__":
  parser = optparse.OptionParser("Usage: %prog [options] target")
  parser.add_option("-t", "--thread", dest = "threads_num",
    default = 1, type = "int",
    help = "[optional]number of  theads,default=10")
  (options, args) = parser.parse_args()
  if len(args) < 1:
    parser.print_help()
    sys.exit(0)

  print "+----------------+------+--------------------+------------------------------+"
  print "|     IP         |Status|       Server       |            Title             |"
  print "+----------------+------+--------------------+------------------------------+"

  s = scan(cidr=args[0],threads_num=options.threads_num)
  s.run()

linux命令——touch、cp命令

发表于 2016-08-13 | 分类于 操作系统 |

今天来写一下关于文件处理的两个命令。

创建文件——touch命令

有时我们可能需要创建一个空文件，比如某些程序要求在运行前要求在目录中有某个文件，这是我们就可以使用touch命令。touch命令将创建一个指定的大小为0的新文件，并将当前用户设置为文件的所有者。

基本格式

touch -选项 文件名

其他用途

touch也可以用来修改文件的访问时间和修改时间。

touch 文件名 #修改访问时间和修改时间
touch -a 文件名 #只修改访问时间
touch -m 文件名 #只修改修改时间
touch -t 时间戳 文件名 #使用时间戳指定的时间,时间戳为形如201608132259的字符串,时间戳可以为未来的时间

复制文件——cp命令

在windows我们经常在图形化窗口下进行将一个文件从一个目录位置复制到另一个目录位置的操作，在linux中我们可以使用cp命令来实现类似操作。

基本格式

cp -选项 源文件 目标文件 #源文件,目标文件的目录可以用绝对路径或相对路径表示;文件名可以使用通配符
例如: 
cp test /home/lemur/test1 #将当前目录下的test文件复制到/home/lemur目录下，新文件的名称为test1
cp test /home/lemur #将当前目录下的test文件复制到/home/lemur目录下，新文件的名称为test
cp /home/lemur/test . #将/home/lemur/test复制到当前目录下
默认情况下，新复制的文件是一个新文件，相比源文件，有新的访问时间和修改时间，有新的索引节点。 ## 常用参数 
-a 归档文件，保留其属性
-f 强制覆盖目标文件，没有提示
-l 创建硬链接，而不是覆盖文件
-s 创建符号链接，而不是覆盖文件
-p 如果可能保留文件属性，可以使文件的访问时间和修改时间与源文件一致
-r 递归复制文件
-R 递归复制目录
-u 仅在源文件比目标文件的修改日期新时复制文件(更新)

linux命令——cd、ls命令

发表于 2016-08-13 | 分类于 操作系统 |

cd命令

linux中使用cd命令在不同目录之间转移，基本格式为：

cd destination
若不指定destination，则会返回主目录。 ## 相对文件路径与绝对文件路径 ### 绝对文件路径 绝对文件路径定义文件在虚拟目录中的确切位置，以虚拟目录的根目录开头，是文件的完整名称，就像绝对海拔。因此要引用usr目录中的lib目录中的apache目录，就可以使用绝对文件路径： 
/usr/lib/apache
然而，如果要从apache目录转到上级lib目录，就不得不输入如下命令，这是很繁的： 
cd /usr/lib

相对文件路径

相对文件路径则是相对当前的位置指定目标文件路径，即使用以相对当前位置的偏移量确定目标文件。相对文件路径的开头不是表示根目录的正斜杠’/‘，而是以”.”或”..”两个特殊字符。其中”.”字符表示当前目录，”..”字符表示父目录。例如,若使用相对文件路径从apache目录转到上级lib目录,可使用如下命令:

cd ..
若要从apache目录转到上级lib目录中的php目录，可输入如下命令: 
cd ../php
若要从apache目录转到apache目录中的test目录，可输入如下命令: 
cd ./test

ls命令

基本格式

ls命令用来查看目录中的文件和目录，其基本格式为:

ls [-options] [directory]

若不指定directory,则会列出当前目录下的目录和文件。

常用命令参数

-a 显示所有文件，包括隐藏的文件(文件名以.开头的)
-F 标识出可执行文件
-i 显示每个文件的索引节点(inode)
-l 以长列表格式显示
-R 递归地列出子目录和文件
-s 打印每个文件的块大小

长列表格式每一列的信息分别为:文件类型(比如目录d,文件-,字符设备c,块设备b,网络设备n),文件权限,指向该文件的硬链接数量,文件所属用户的名称,文件所属组的名称,文件的字节大小,文件的修改时间,文件名称。

常用命令参数组合

当你想要指定多个参数选项时,linux允许你将多个参数选项略去”-“连在一起书写为一个字符串。比较常见的组合是:-sail。其会在长列表格式输出的基础上输出文件的块大小和索引号。索引节点是文件的一个独特的标识数，是文件创建时由内核分配的(内核分配给虚拟目录中的每个对象)。

过滤列表输出

默认情况下,ls命令会输出指定目录中的所有目录和文件，但有时候这显得有些多余，尤其是在只需要查看某一个文件的信息时，这时可以使用过滤器，指定输出哪些文件或目录的信息。如果将特定文件的名称指定为过滤器，则ls命令将只显示该文件的信息。过滤器的工作方式类似于一个简单的文本匹配字符串，在命令行参数之后包含过滤器的方式如下:

ls -l test

这样ls命令将只显示test文件的信息。
有时可能不知道要查找的文件的确切名称或者想列出所有名称包含某一段特定字符串的文件，可以使用通配符，ls命令支持标准的通配符：”?”表示一个字符，”*”表示零个或多个字符。

linux命令--目录结构

发表于 2016-08-11 | 分类于 操作系统 |

今天写一下linux的目录结构。

linux目录与windows的不同之处

linux的路径名中不使用驱动器盘符

在windows中，PC上安装的物理驱动器决定了文件的路径名。windows为每个物理驱动器分配一个盘符。每个物理驱动器包含自己的目录结构，以便访问存储的文件。例如在windows中经常看到如下形式的文件路径:

C:\docunments and settings\turing\test.txt
该路径表示文件"test.doc"存储在目录"turing"中，而该目录又位于"docunments and settings"目录中，而"docunments and settings"位于盘符为C的物理驱动器或称硬盘分区(通常是PC的第一个硬盘)上。windows形式的路径可以告诉你文件所在的确切物理驱动器。同时应该注意到windows形式的路径使用的是反斜杠"\"。 在linux中，则是使用一个目录结构存储所有物理驱动器中的文件。例如在linux中经常看到如下形式的文件路径: 
/home/user/.bashrc
该路径表示文件".bashrc"存储在目录"user"中，而"user"又位于目录"home"中，路径不包含有关文件位于哪个物理驱动器的任何信息。文件名中的"."表示该文件是一个隐藏文件。 ## linux目录结构 linux的目录称为**虚拟目录**，文件系统将第一个硬盘称为**根驱动器**，根驱动器中的目录称为**根目录**，虚拟目录中的其他目录都是从根驱动器中的根目录展开的，就像树的枝叶是从树干生长出的一样。虚拟目录中用于分配其他存储器的目录称为**挂载点**。虚拟目录使文件和目录都出现在挂载点目录中，即使它们存储在另一个驱动器中。系统文件一般存储在根驱动器中，其他用户文件存储在其他驱动器中。 ## 通用的linux目录名称 
/ 虚拟目录的根目录，通常此处没有文件
/bin 二进制文件目录，存储GNU实用程序(用户级别)
/boot 引导目录，存储引导文件
/dev 设备文件目录，在此文件中包含设备文件目录(linux将设备看作一个个特殊文件，称为设备文件)
/etc 系统配置文件目录，例如存储系统账户信息的"passwd"文件就位于此目录下
/home 主目录，各用户(除根用户)目录默认在此目录中
/lib 库目录，存储系统和应用程序库文件
/media 媒体目录，可移动设备常用的挂载点
/mnt 挂载目录(mount)，另一个可移动设备常用的挂载点
/opt 可选目录，常用于存储可选文件包，一些软件的默认安装目录
/root 根主目录(或许叫根用户的用户目录更加合适)
/sbin 系统二进制目录，存储GNU实用程序(管理级别)
/tmp 可以在此目录中创建和销毁临时工作文件
/usr 用户安装软件的目录，另一些软件的默认安装目录，类似于windows中的"program files"目录
/var 可变目录，用于存储经常更改的文件，例如日志文件
